# AVL+ verifier Scala-oracle parity vectors

`scala_avl_vectors.json` pins the AUTHORITATIVE Scala scrypto `BatchAVLVerifier`
outcome for a set of `(startingDigest, proof, op)` triples: whether
`performOneOperation` succeeds and the resulting `digest`. Consumed by
`ergo-sigma/tests/avl_scala_oracle_parity.rs`, which asserts the Rust
`ergo_sigma::avl::AvlVerifier` agrees:

- **Scala success** ⇒ Rust op `Ok` AND `digest()` equals Scala's digest
  (the reject-valid / accept-invalid guard — a proof Scala accepts must not make
  the Rust crate panic into `Err`).
- **Scala failure** ⇒ Rust op fails (fail-closed parity for the op-time panic
  class — wrong-op replay, off-path removes — and malformed/truncated proofs).

## Provenance (oracle, per CLAUDE.md §3)

Generated by the Scala reference implementation, NOT by the Rust crate (a Rust
self-oracle would only prove consistency, not parity). The proofs are produced by
the Scala `BatchAVLProver`, so this is a true verifier differential with no
dependence on Rust-prover/Scala-prover byte parity — and it mirrors production,
where the Rust node verifies Scala-produced AVL proofs in mainnet blocks.

- Oracle source: `scorex.crypto.authds.avltree.batch.{BatchAVLProver,BatchAVLVerifier}`
  via `org.scorexfoundation::sigma-state:6.0.2` (same dependency the existing
  `scripts/jvm_serde_oracle` uses).
- Generator: `scripts/avl_oracle/AvlOracle.scala`.

## Regenerate

```sh
scala-cli run scripts/avl_oracle/AvlOracle.scala \
  > test-vectors/ergo-sigma/avl-proof-parity/scala_avl_vectors.json
```

(scrypto logs the expected reconstruction failures of the malformed-proof
vectors to stderr; those are benign — the `Try` still returns `Failure`. Only
stdout is captured.)

## Coverage (106 vectors: 52 accept / 54 reject)

- valid lookup / remove across tree sizes 1..16 (exercises rebalancing/deleteMax
  on correct proofs — the accept side that closes the reject-valid gap)
- valid insert / update
- **wrong-op replay**: a valid lookup proof used for a remove (the Rust op-time
  panic class) ⇒ Scala fails closed
- remove of an absent key off the witnessed path ⇒ Scala failure
- fixed value-length (`valueLengthOpt = Some(4)`) variants
- malformed-but-constructible proofs (truncated, byte-flipped) ⇒ Scala failure
