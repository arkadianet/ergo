# Compile-only probe sources — M3 Task-11 wave-1 GraphBuilding reject-gate
# vectors (lib.rs D-C5). Unlike golden_seed.txt records these carry no typer
# s-expression: they exist solely to feed the compile oracle, so the
# compile_seed.json recapture (`compile_seed_live_recapture`) replays them
# through the SAME cc/cce/ccs verbs and records the verdict verbatim — no
# oracle field is ever hand-written.
#
# Format: verb<TAB>tree_version<TAB>source. The recapture spawns one oracle
# process per distinct tree_version (ORACLE_TREE_VERSION), ORACLE_NETWORK=
# testnet throughout.
#
# Selection rule: REJECT probes (the wave-1 gates are all reject-direction)
# plus the ACCEPT fold-boundary controls whose reductions AGREE on both
# sides. ACCEPT boundaries whose trees are unevaluable on OUR side (the
# D-C4 multi-arg-FuncValue class: unused/fold-slot lambdas) are deliberately
# NOT committed — the semantic gate would flag them mixed Ok/Err; they are
# pinned verdict-only in tree.rs unit tests instead.
#
# ── fix 1: bit ops — Scala GraphBuilding has no BitOp/BitInversion lowering ──

cc	3	sigmaProp((1 & 2) == 0)
cc	3	sigmaProp((5 ^ 3) == 6)
cc	3	sigmaProp((5 << 1) == 10)
cc	3	sigmaProp((5 >> 1) == 2)
cc	3	sigmaProp((5 >>> 1) == 2)
cc	3	sigmaProp((~1) == -2)
cc	3	sigmaProp((1 | 2L) == 3L)
ccs	3	sigmaProp((b1 | b2) == 3.toByte)
# ── fix 2: zero-arg lambdas + non-1-arg applications ──
cc	3	{ val f = {(x: Int, y: Int) => x + y}; sigmaProp(f(1, 2) == 3) }
cc	3	{ val f = {(x: Int, y: Int, z: Int) => x + y + z}; sigmaProp(f(1, 2, 3) == 6) }

cc	3	{ val f = {() => 1}; sigmaProp(f() == 1) }
cc	3	{ val unused = {() => 1}; sigmaProp(true) }
cc	3	sigmaProp({(x: Int, y: Int) => x + y}(1, 2) == 3)
# ── fix 3: function-typed lambda parameters ──
cc	3	{ val h = {(f: Int => Int) => f(10)}; sigmaProp(h({(x: Int) => x + 1}) == 11) }
